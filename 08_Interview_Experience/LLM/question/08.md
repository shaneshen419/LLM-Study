## agentic 长文本训练时候容易 oom，你有什么好的优化吗？ 

Agentic 长文本训练（涉及多轮对话、长 Tool Call 历史、大规模 RAG 检索）确实是显存杀手。由于 Self-Attention 的计算复杂度是序列长度的平方 ( $O(n^2)$ )，且 KV Cache 和激活值（Activations）随长度线性增长，OOM 是非常普遍的挑战。

### 一、核心算子优化（必须开启）

- **FlashAttention-2/FlashAttention-3**：
    - **原理**：通过 IO 感知分块计算 Attention，不存储完整的 $O(n^2)$ 注意力矩阵。
    - **效果**：极大地降低显存占用，且序列越长加速效果越明显。这是长文本训练的基石。
- **Liger Kernel（Fused Kernels）**：
    - **原理**：针对长文本中开销巨大的 RMSNorm、RoPE、Softmax 和 CrossEntropy 进行算子融合（Triton实现）。
    - **效果**：尤其是在长序列下，能够减少中间变量的显存驻留，通常能多塞下 20%-40% 的序列长度。

### 二、分布式策略优化（架构层面）
- **DeepSpeed ZeRO-3 + Offload**：
    - **原理**：ZeRO-3 将参数、梯度和优化器状态分布到所有 GPU 上。
    - **Offload**：如果 GPU 显存实在不够，将优化器状态和参数卸载到 **CPU内存** 甚至 NVMe 硬盘上。
- **序列并行（SP）**：
    - **原理**：这是解决长文本 OOM 的核武器。它将**单个序列**切分成多份，分布在不同的  GPU 上计算。
    - **常用方案**：
        - **DeepSpeed-Ulysses**：通过 ALL-to-ALL 通常在Attention计算前后切分序列，通信开销较小，非常适合 Agentic 场景。
        - **Megatron-SP**：在 LaterNorm 和 Dropout 环节切分激活值。
- **激活值检查点**：
    - **建议**：开启 selective 或 full 检查点。它不存储前向传播的激活值，而是在反向传播时重新计算。虽然计算量增加 33%，但显存占用会从 $O(n)$ 降至 $O(\sqrt n)$ 。

### 三、针对 Agentic 场景的特殊策略
- **Reference Model Offloading（针对DPO/GRPO）**：
    - 在训练Agent 的偏好对齐时，Reference Model 只需要做前向推理。可以将其放在 CPU 上，或者使用 **4-bit量化** 版本作为参考模型，这能省下巨大的显存空间。
- **Prefix Caching（训练版）**：
    - Agent 训练中往往有很长的 System Prompt 或 Tool 定义。在 DataLoader 层面确保这些公共前缀的一致性，结合一些支持计算复用的框架，可以减少重复计算带来的开销。
- **Packing without Padding (变长序列打包)**：
    - Agent 数据的长度差异很大。不要使用 Padding，而是使用 flash_attn_varlen_func（如微调框架 unsloth 或 axolotl 所做的），将多个短样本拼接成一个固定长度的长样本，消除无效计算。

### 四、显存节省的微调技巧

- **LoRA / QLoRA**：
    - 如果全量微调（Full Fine-tuning）必然 OOM，考虑使用 LoRA。只训练低秩矩阵，显存占用会减少 80% 以上。
- **长文本分级策略**：
    - **两阶段训练**： 先在短文本上微调 Agent 逻辑，再在第二阶段开启序列并行（SP）进行长文本能力的迁移。这样可以避免全程背负巨大的显存压力。

### 五、硬件与底层调优
- **启用 BF16 而非 FP16**：
    - BF16 具有与 FP32 相同的动态范围，在长文本训练中更不容易出现梯度溢出，且比 FP32 省一半显存。
- **调整 CPU 内存管理**：
    - 如果开启了 Offload，确保主机的 Pinned Memory 足够大，以加速 CPU 与 GPU 之间的数据交换。

### 总结建议的配置组合：
1. 入门级： LoRA + FlashAttention-2 + Gradient Checkpointing。
2. 专业级： 全量微调 + DeepSpeed ZeRO-3 + Liger Kernel + BF16。
3. 极长文本 (32k+)： DeepSpeed ZeRO-3 + Sequence Parallelism (Ulysses) + FlashAttention-2。
