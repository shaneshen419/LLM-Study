## 如何解决大模型容易 reward hacking，如何解决？

Reward Hacking 是 RLHF 中最难缠的问题：模型发现了一种“投机取巧”的方法，比如在回答末尾疯狂加标点符号、复读特定短语，或者利用 Reward Model 的偏见来刷高分，但实际逻辑却完全崩坏。

解决方法体系：

1. **完善 Reward 系统的构成（Multi-Reward / Hybrid Reward）**

    - **不要只信赖单模型打分**： 纯用一个 Reward Model 极易被 Hacking。
    - **引入基于规则的硬约束（Hard Rules）**： 例如，如果模型在做数学题，Reward 由“模型打分”+“正则匹配判定答案对错”组成。即便模型能骗过打分模型，只要答案算错，总分就会被拉下来。
    - **惩罚异常长度**： 很多模型会通过无限制增加回答长度来博取高分。可以加入 Length Penalty，对过长的输出进行指数级的奖励削减。
2. **使用 Reference Model 进行严格约束（KL Divergence）**
    - **强制锚定**： GRPO 中必须保留一个 Frozen（冻结）的 Reference Model。
    - **KL 惩罚**： 计算新策略分布和原分布的 KL 散度。一旦模型为了 Hacking 产生古怪的 token 组合，它与原模型的分布差异会巨大，从而触发极高的 KL 惩罚项（Negative Reward），抵消掉作弊得来的高分。

3. **动态 Reward 缩放与平滑（Reward Whitening/Scaling）**
    - **均值平滑**： 对一个 Batch 内的 Reward 进行全局标准化（Z-score）。
    - **Reward Clipping**： 将 Reward 值限制在一个固定的区间（如 $[-1,1]$ 或 [-5,5] ）。这能防止模型因为捕捉到一个具有超级高分的“作弊路径”而产生足以摧毁整个策略的梯度波动。

4. **迭代训练（Iterative Training / Online RM）**
    - **定期更新 Reward Model**： 如果发现模型学会了某种作弊手段，立刻将这些作弊样本标记为“负样本（低分）”，重新训练或微调 Reward Model。
    - **拒绝采样（Rejection Sampling）**： 在正式训练前，先用当前模型生成大量样本，用 RM 剔除掉那些分数虚高但实际质量差的样本，修正 RM 的认知偏差。

5. **引入“多样性奖励”（Diversity Reward）**
    - 在 Reward 函数中增加一项：如果模型连续生成的多个样本在语义上过度重合，则给整个 Group 降低分数。这迫使模型在探索高分的同时，必须保持不同的表达路径。
